# INSIGHT FOOTBALL Visual Production System v1.0

Status: Official visual production standard  
Purpose: define exactly what an INSIGHT FOOTBALL video should look, feel, sound, and move like.

This is not implementation code. This is the visual production language for editors, motion designers, automation builders, and template-rendering systems.

---

## 1. Visual Identity

INSIGHT FOOTBALL should look like a clean football intelligence dashboard built for short-form football fans.

The visual identity should feel:

- Clean, modern, and sharp.
- Fast but not chaotic.
- Smart but not nerdy.
- Premium but simple.
- Football-first, not generic social media.
- Built for fans who want the story before kickoff.

The visuals must always support the story. They must never become the product by themselves.

### Primary Visual Mood

Clean football intelligence:

- Dark pitch-inspired backgrounds.
- Strong team badges.
- Bold questions.
- Simple dashboard cards.
- Fast, readable data moments.

### Secondary Mood

Matchday tension:

- Subtle stadium light.
- Soft pitch texture.
- Clock/timer graphics.
- Movement that feels like buildup before kickoff.

### Background Style

Use:

- Dark pitch texture.
- Subtle grass lines.
- Abstract stadium floodlight glow.
- Soft moving gradient only as a background layer, not as the main identity.
- Pitch diagrams when explaining shape, pressure, or movement.

Avoid:

- Busy stadium photos behind text.
- Random stock footage.
- Bright cluttered backgrounds.
- Broadcast match footage unless licensed.

### Logo Placement

Use the INSIGHT FOOTBALL logo or wordmark:

- Opening scene: centered or top-center, large enough to read.
- Main video scenes: small watermark in top-right or top-left safe area.
- Dashboard scene: small watermark near header.
- Closing CTA: visible but not louder than the debate question.

Do not place the logo over captions or key data.

### Brand Watermark Rules

- Keep watermark visible but quiet.
- Use 70-85% opacity.
- Position inside platform safe area.
- Do not animate the watermark constantly.
- Do not let the watermark compete with team badges.

### Use of Club Colors

Club colors may be used as accents only:

- Badge glow.
- Comparison bars.
- Small side labels.
- Team name underline.

Club colors must not replace INSIGHT FOOTBALL's core visual palette. The video should still feel like INSIGHT FOOTBALL even when the teams change.

### Dark Mode vs Light Mode

Default mode:

- Dark mode.

Use dark mode because:

- It feels like a matchday dashboard.
- It keeps badges and data cards sharp.
- It works well on mobile.
- It handles club color accents cleanly.

Light mode may be used only for:

- Special template variants.
- Very bright club identity moments.
- Data card inserts where contrast remains strong.

### Pitch Graphics

Use pitch graphics for:

- Tactical moments.
- Pressure direction.
- Passing route.
- Defensive shape.
- Key player zone.

Pitch graphics should be simple:

- 2-4 player markers.
- 1-3 arrows.
- One short label.

Avoid tactical diagrams that look like coaching manuals.

### Icons

Use icons to simplify meaning:

- Timer: early pressure.
- Arrow: movement or momentum.
- Shield: defensive strength.
- Spark: X-factor.
- Warning triangle: risk or uncertainty.
- Ball: goal threat.
- Comment bubble: CTA.

Icons should be consistent in line weight and color.

### Player Images

Use player images only when legally safe.

Allowed:

- Licensed image library.
- Club-approved assets.
- Owned cutouts.
- Public-domain or properly licensed assets.

Avoid:

- Random web images.
- Screenshots from broadcasts.
- Images with unclear rights.

If rights are unclear, use:

- Player name card.
- Silhouette.
- Team badge.
- Shirt number-style graphic.

### Team Badges

Team badges are core assets.

Rules:

- Show both badges clearly on match card and CTA.
- Use one badge in story-specific moments if one team is being discussed.
- Do not distort badge aspect ratios.
- Do not crop badges aggressively.
- Use licensed or approved badge files.

### Stadium Images

Use stadium images only when:

- Licensed.
- Clean enough for text overlay.
- Visually relevant to home advantage or atmosphere.

If not available, use a generic stadium-light or pitch background.

### Data Cards

Data cards should feel like visual proof, not spreadsheet screenshots.

Rules:

- One idea per card.
- Maximum two numbers per card.
- Short label.
- Short explanation.
- High contrast.
- 8px or less corner radius.

## 2. Color System

Use a tight palette that works for football, dashboards, and vertical video.

| Role | Name | Hex | Usage |
|---|---|---|---|
| Primary brand color | Whistle Green | `#18D26E` | Insight highlights, active bars, key phrases. |
| Secondary color | Floodlight Cyan | `#38BDF8` | Data accents, lines, secondary highlights. |
| Accent color | Match Gold | `#F5C542` | Surprising fact, X-factor, key moment. |
| Background color | Night Pitch | `#07110D` | Main background. |
| Surface color | Graphite Card | `#111827` | Dashboard cards and panels. |
| Text color | Clean White | `#F8FAFC` | Main text. |
| Muted text color | Touchline Grey | `#94A3B8` | Labels and secondary text. |
| Warning/risk color | Pressure Red | `#F43F5E` | Risk, uncertainty warnings, injury alerts. |
| Positive indicator | Advantage Green | `#22C55E` | Positive form, edge, momentum. |
| Neutral indicator | Neutral Slate | `#64748B` | Draw, uncertainty, balanced comparison. |

### Color Usage Rules

- Use Night Pitch as the default background.
- Use Clean White for main readable text.
- Use Whistle Green for brand insight moments.
- Use Match Gold only for surprise or X-factor.
- Use Pressure Red sparingly for risk, injury, or warning.
- Do not use more than three accent colors in one scene.

### Club Color Rules

Club colors may appear in:

- Badge glow.
- Team-side comparison bars.
- Small underline.
- Score/edge markers.

Club colors may not:

- Replace the full background.
- Make captions hard to read.
- Clash with dashboard readability.
- Turn every video into a club-color template.

If club colors clash, use badges only and keep the INSIGHT FOOTBALL palette dominant.

## 3. Typography System

Typography must be readable on a phone in one glance.

### Headline Font Style

Use:

- Bold condensed sans serif.
- Strong uppercase or title case.
- Clean sports-media feel.

Suggested styles:

- Bebas Neue.
- Anton.
- Oswald Bold.
- League Spartan Bold.
- Montserrat ExtraBold.

Headline use:

- Main question.
- Surprising fact.
- CTA.

### Body Font Style

Use:

- Clean sans serif.
- Medium or semibold weight.

Suggested styles:

- Inter.
- Montserrat.
- Manrope.
- Source Sans 3.

### Caption Font Style

Use:

- Bold or semibold sans serif.
- High contrast.
- Simple shapes.

Captions must not be thin or decorative.

### Number/Stat Font Style

Use:

- Tabular number style if available.
- Bold.
- Slightly larger than labels.

Numbers should feel like dashboard readings, not spreadsheet cells.

### Dashboard Label Style

Use:

- Small uppercase labels.
- Muted grey.
- Short labels only.

Examples:

```text
MATCH EDGE
FORM
KEY ADVANTAGE
UNCERTAINTY
X-FACTOR
```

### Maximum Text Per Screen

General rule:

- Main headline: 3-8 words.
- Secondary text: 5-12 words.
- Caption: max 7 words per line.
- Dashboard: max 6 cards total.
- Evidence card: one stat or claim.

### Font Weight Rules

- Main headline: 800-900.
- Captions: 700-800.
- Labels: 600.
- Supporting text: 500-600.

### Text Hierarchy

1. Main story text.
2. Team badges or team names.
3. Evidence number or chart.
4. Caption.
5. Brand watermark.

### Mobile Readability Rules

- No tiny labels below 28px on 1080x1920.
- Captions should generally be 48-64px.
- Headlines should generally be 80-140px.
- Dashboard labels should be readable without pausing.
- Avoid text near platform UI edges.

## 4. Standard 60-Second Video Structure

Every match preview follows this visual timing.

| Time | Segment | Purpose |
|---|---|---|
| 0-3s | Brand opening | Establish the promise. |
| 3-10s | Surprising fact / hidden match angle | Stop the scroll. |
| 10-18s | Central question | Give viewers the problem to follow. |
| 18-42s | Story explanation with evidence | Make the story simple and believable. |
| 42-55s | Insight dashboard / supporting evidence | Summarize the edge visually. |
| 55-60s | Final answer and engagement question | Invite comments. |

### 0-3s: Brand Opening

Purpose:

- Tell viewers what kind of video this is.

Visual style:

- INSIGHT FOOTBALL wordmark.
- Dark pitch background.
- Both team badges small or blurred behind.

On-screen text:

```text
Before the first whistle
```

Motion style:

- Slow zoom in.
- Small logo reveal.

Caption style:

```text
Here's the insight.
```

Audio cue:

- Short intro sting.
- Low, clean impact.

Common mistakes:

- Long logo animation.
- Too much branding.
- No match context.

### 3-10s: Surprising Fact / Hidden Match Angle

Purpose:

- Hook the viewer with a real angle.

Visual style:

- Big text.
- One team badge.
- Timer, arrow, or warning icon.

On-screen text:

```text
The first 20 minutes matter
```

Motion style:

- Quick punch-in.
- Text pop.
- Small icon pulse.

Caption style:

- Short and compressed.

Audio cue:

- Soft hit or whoosh.

Common mistakes:

- Revealing a generic fact.
- Using too many numbers.
- Calling the angle a prediction.

### 10-18s: Central Question

Purpose:

- Show the one question the video answers.

Visual style:

- Large question text.
- Split team badges.
- Pitch or stadium background.

On-screen text:

```text
Can Arsenal survive the fast start?
```

Motion style:

- Stable with slight push-in.

Caption style:

- Match the question closely.

Audio cue:

- Music dips slightly to let question breathe.

Common mistakes:

- Asking multiple questions.
- Making the question too technical.
- Hiding the question in small text.

### 18-42s: Story Explanation With Evidence

Purpose:

- Explain one idea in normal football language.

Visual style:

- Evidence card.
- Team comparison.
- Simple pitch graphic.
- X-factor card if needed.

On-screen text:

- One claim at a time.

Motion style:

- Slide transitions.
- Bar movement.
- Pitch arrows.

Caption style:

- Always visible.
- No paragraph captions.

Audio cue:

- Light transitions between evidence points.

Common mistakes:

- Too many stats.
- Too much tactical diagram detail.
- Showing visuals unrelated to voiceover.

### 42-55s: Insight Dashboard / Supporting Evidence

Purpose:

- Summarize the story visually.

Visual style:

- Dashboard cards.
- Probability bars.
- Key advantage card.
- Uncertainty note.

On-screen text:

```text
Small edge: Liverpool
Edge, not guarantee
```

Motion style:

- Dashboard reveal from top to bottom.
- Bars fill from zero.

Caption style:

- Explain naturally, not label-by-label.

Audio cue:

- Dashboard reveal sound.

Common mistakes:

- Making dashboard look like betting odds.
- Too many cards.
- Narrator reading every label.

### 55-60s: Final Answer and Engagement Question

Purpose:

- Give a soft conclusion and invite comments.

Visual style:

- Both team badges.
- Big debate question.
- Comment icon.
- Brand watermark.

On-screen text:

```text
Do you agree?
```

Motion style:

- Settle or slight zoom out.
- Comment icon pulse.

Caption style:

```text
Tell us below.
```

Audio cue:

- Gentle ending hit.

Common mistakes:

- Ending without a question.
- Sounding certain.
- Hard sell.

## 5. Scene Library

## Scene A: Brand Opening

Objective:

- Introduce the brand and promise quickly.

Duration:

- 2-3 seconds.

Visual layout:

- Centered INSIGHT FOOTBALL wordmark.
- Dark pitch background.
- Optional faint match badges behind wordmark.

On-screen text rules:

- Use the signature line.
- Maximum 5 words visible at once.

Animation rules:

- Logo fades or slides in.
- Slow zoom in.
- No long intro.

Required assets:

- Logo or wordmark.
- Pitch background.

Optional assets:

- Team badges.
- Intro light sweep.

Example use:

```text
Before the first whistle... here's the insight.
```

## Scene B: Surprising Fact

Objective:

- Hook viewers with the strongest hidden match angle.

Duration:

- 4-7 seconds.

Visual layout:

- Big fact text in center.
- Team badge on one side.
- Icon near the key phrase.

On-screen text rules:

- One fact only.
- Use no more than one number if possible.

Animation rules:

- Quick punch-in.
- Key number count-up.
- Small icon pulse.

Required assets:

- Team badge.
- Icon.
- Text field.

Optional assets:

- Timer.
- Mini bar.

Example use:

```text
Liverpool scored early in 4 of 5
```

## Scene C: Match Card

Objective:

- Show the fixture clearly.

Duration:

- 3-5 seconds.

Visual layout:

- Home badge left.
- Away badge right.
- Competition and kickoff time below.
- Center "vs".

On-screen text rules:

- Team names must be readable.
- Competition smaller than team names.

Animation rules:

- Badges slide in.
- Match title settles.
- Minimal movement.

Required assets:

- Home badge.
- Away badge.
- Competition name.

Optional assets:

- Competition logo.
- Venue.

Example use:

```text
Liverpool vs Arsenal
Premier League | 20:00
```

## Scene D: Central Question

Objective:

- Show the question the video will answer.

Duration:

- 5-8 seconds.

Visual layout:

- Large question text.
- Pitch or stadium background.
- Team badges small.

On-screen text rules:

- One question only.
- No subtitles competing with it.

Animation rules:

- Slow push-in.
- Question line reveals in two beats.

Required assets:

- Background.
- Team badges.

Optional assets:

- Question mark icon.

Example use:

```text
Can Arsenal survive Liverpool's early pressure?
```

## Scene E: Evidence Card

Objective:

- Show one clear piece of supporting evidence.

Duration:

- 4-6 seconds.

Visual layout:

- One stat or claim.
- One short explanation.
- Icon or small chart.

On-screen text rules:

- One evidence idea only.
- Use short labels.

Animation rules:

- Card slides up.
- Number counts up if used.
- Explanation fades in second.

Required assets:

- Card background.
- Icon or chart.

Optional assets:

- Team badge.

Example use:

```text
Fast home starts
Liverpool begin games aggressively
```

## Scene F: Team Comparison

Objective:

- Compare teams simply.

Duration:

- 5-7 seconds.

Visual layout:

- Two columns.
- Home left, away right.
- Badge, label, bar/meter.

On-screen text rules:

- Maximum 3 comparison rows.
- Short labels only.

Animation rules:

- Horizontal slide.
- Bars fill side by side.

Required assets:

- Team badges.
- Bar/meter component.

Optional assets:

- Club color accents.

Example use:

```text
Fast starts vs late control
```

## Scene G: Tactical Moment

Objective:

- Explain a simple football pattern.

Duration:

- 5-8 seconds.

Visual layout:

- Pitch graphic.
- 2-4 player markers.
- 1-3 arrows.
- One label.

On-screen text rules:

- No tactical paragraphs.
- Use plain language.

Animation rules:

- Arrows draw in.
- Player markers move slightly.
- Camera pans slowly if needed.

Required assets:

- Pitch graphic.
- Arrows.
- Player markers.

Optional assets:

- Ball icon.

Example use:

```text
Can Arsenal play through the press?
```

## Scene H: X-Factor Player

Objective:

- Highlight one player who could change the match.

Duration:

- 4-6 seconds.

Visual layout:

- Player name.
- Team badge.
- One simple reason.
- Optional player cutout.

On-screen text rules:

- Do not overload with career stats.
- One reason only.

Animation rules:

- Name slides in.
- Badge glow.
- Optional cutout parallax.

Required assets:

- Team badge.
- Player name.

Optional assets:

- Licensed player cutout.

Example use:

```text
X-factor: Arsenal's first pass out
```

## Scene I: Insight Dashboard

Objective:

- Summarize the conclusion visually.

Duration:

- 8-12 seconds.

Visual layout:

- Header: Match Insight.
- Probability/edge bar.
- Form card.
- Key advantage card.
- Uncertainty card.
- X-factor card.
- Surprising detail card.

On-screen text rules:

- Maximum 6 data points.
- No paragraph text.
- Use plain labels.

Animation rules:

- Reveal top to bottom.
- Bars fill from zero.
- Cards fade in one by one.

Required assets:

- Dashboard template.
- Bars.
- Icons.
- Team badges.

Optional assets:

- Mini pitch diagram.

Example use:

```text
Small edge: Liverpool
Uncertainty: Medium
```

Important:

The voiceover should not mechanically read dashboard labels. It should explain naturally.

## Scene J: Final Question / CTA

Objective:

- Invite comments.

Duration:

- 3-5 seconds.

Visual layout:

- Big debate question.
- Both team badges.
- Comment icon.
- Brand watermark.

On-screen text rules:

- One question.
- One CTA.

Animation rules:

- Slight zoom out or settle.
- Comment icon pulse.

Required assets:

- Team badges.
- Comment icon.
- Brand watermark.

Optional assets:

- Outro sound.

Example use:

```text
Do you agree?
Tell us below.
```

## 6. Motion Language

Motion should feel controlled, repeatable, and useful.

| Motion Type | When To Use | When Not To Use | Speed | Purpose | Example |
|---|---|---|---|---|---|
| Slow zoom | Opening, central question, CTA | Data-heavy scenes | 2-5% scale over scene | Builds focus | Logo slowly pushes in. |
| Quick punch-in | Surprising fact | Long explanation | 4-8 frames | Creates emphasis | Fact text snaps closer. |
| Slide transition | Evidence and comparisons | Every single cut | 8-14 frames | Moves story forward | Evidence card slides up. |
| Bar animation | Dashboard, team comparison | Non-data scenes | 0.5-1s fill | Makes numbers readable | Edge bar fills to 43%. |
| Number count-up | One key stat | Many stats at once | 0.5-0.8s | Adds energy to data | 4 of 5 counts up. |
| Line reveal | Central question, labels | Tiny text | 0.4-0.7s | Makes text feel intentional | Question reveals in two lines. |
| Pitch arrow movement | Tactical moment | Non-tactical claims | 0.8-1.2s | Shows football pattern | Arrow from midfield to wide zone. |
| Badge entrance | Match card, CTA | Dashboard labels | 0.5-0.8s | Establishes teams | Badges slide in from sides. |
| Text pop | Hook, CTA | Long captions | 0.2-0.4s | Adds emphasis | "Fast start?" pops. |
| Dashboard reveal | Dashboard scene | Hook scenes | 1-2s | Organizes summary | Cards reveal top to bottom. |

No random effects. Every movement must help viewers understand the story.

## 7. Camera Movement Rules

| Scene Type | Camera Movement | Rule |
|---|---|---|
| Brand Opening | Slow zoom in | Make the brand feel immediate. |
| Surprising Fact | Punch-in | Hit the key idea fast. |
| Match Card | Stable | Let viewers read teams and kickoff. |
| Central Question | Slow push-in | Give the question weight. |
| Evidence Card | Slight slide or stable | Do not distract from the stat. |
| Team Comparison | Horizontal slide | Reinforce side-by-side contrast. |
| Tactical Moment | Slight pitch pan | Follow the arrow or key movement. |
| X-Factor Player | Subtle parallax or stable | Keep focus on name and reason. |
| Dashboard | Top-to-bottom reveal | Make data easy to scan. |
| Final CTA | Settle or slight zoom out | Give viewers space to respond. |

Avoid:

- Excessive shakes.
- Too many transitions.
- Random spinning.
- Fast movements that make text hard to read.
- Moving captions.

## 8. Caption System

Captions are mandatory.

### Caption Rules

- Always visible.
- Short.
- Maximum 7 words per caption line.
- 1-2 lines maximum.
- Readable on mobile.
- May compress the voiceover slightly.
- Important words can become bigger kinetic text.
- Do not cover important graphics.
- Avoid full paragraph captions.

### Caption Placement

Default:

- Lower third.
- Above platform UI safe area.
- Center aligned.

Do not place captions:

- Over team badges.
- Over dashboard bars.
- At the very bottom edge.
- Behind app buttons.

### Caption Size

Recommended for 1080x1920:

- 48-64px for captions.
- 70-110px for kinetic key phrases.

### Caption Background

Use:

- Semi-transparent dark strip.
- Soft shadow.
- No heavy box unless background is busy.

### Highlight Rules

Highlight only:

- Team names.
- Key phrase.
- Surprise word.
- Debate question.

Do not highlight every line.

### Timing Rules

- Captions should appear with the spoken phrase.
- Keep each caption on screen long enough to read.
- Do not change captions faster than viewers can process.

### Voiceover-to-Caption Examples

Voiceover:

```text
Liverpool are not just winning; they are creating better chances every week.
```

Caption:

```text
Liverpool create better chances.
```

Voiceover:

```text
The question is simple: can Arsenal survive Liverpool's fast start?
```

Caption:

```text
Can Arsenal survive?
```

Voiceover:

```text
The edge goes to Liverpool, but football can still surprise us.
```

Caption:

```text
Edge, not guarantee.
```

## 9. Dashboard Design

The INSIGHT FOOTBALL dashboard is visual support, not the story.

### Required Dashboard Items

1. Match edge / likely outcome.
2. Form comparison.
3. Key advantage.
4. Uncertainty level.
5. X-factor.
6. Surprising detail.

### Dashboard Layout

Recommended 9:16 layout:

```text
[Header: Match Insight]
[Team badges + fixture]
[Probability / edge bar]
[Form comparison card]
[Key advantage card]
[Uncertainty + X-factor cards]
[Surprising detail footer]
```

### Cards

Rules:

- 8px or less border radius.
- Dark surface.
- Thin accent line.
- Label at top.
- Main value in bold.
- One sentence maximum.

### Bars

Use bars for:

- Match edge.
- Team comparison.
- Form meter.

Rules:

- Use whole numbers.
- Avoid decimal precision.
- Label as edge, not guaranteed result.

### Icons

Use icons to reduce reading:

- Shield: defensive note.
- Arrow: tactical advantage.
- Timer: early pressure.
- Spark: X-factor.
- Warning: uncertainty.
- Chart: dashboard.

### Animation Sequence

1. Header appears.
2. Team badges settle.
3. Edge bar fills.
4. Form card appears.
5. Key advantage card appears.
6. Uncertainty and X-factor appear.
7. Surprising detail appears last.

### Maximum Data Points

Maximum:

- 6 dashboard items.
- 3 probability values.
- 2 team comparison rows.
- 1 X-factor.
- 1 uncertainty note.

### Text Allowance

Each card:

- Label: 1-3 words.
- Main value: 1-5 words.
- Explanation: 6-10 words max.

### Simplifying Complicated Stats

Use plain replacements:

| Complicated Stat | Simplified Visual |
|---|---|
| Expected goals differential | Better chances |
| Transition efficiency | Faster breaks |
| PPDA / pressing intensity | Presses higher |
| xGA | Chances allowed |
| Field tilt | Territory |

### Showing Uncertainty Without Betting Tone

Use:

```text
Small edge
Could matter
Not a guarantee
Football can still surprise us
Medium uncertainty
```

Avoid:

```text
Lock
Banker
Guaranteed
Safe bet
Must win
```

Narration example:

```text
Everything we looked at gives Liverpool the edge, but Arsenal still have enough quality to punish one mistake.
```

## 11. Audio and Sound Design

### Voiceover Style

The voice should sound:

- Conversational.
- Confident.
- Friendly.
- Curious.
- Human.
- Not robotic.
- Not like a betting tipster.

Voice direction:

```text
Talk like a friend who knows football, not a presenter reading a betting preview.
```

### Voice Speed

Recommended:

- 130-155 words per minute.
- 120-145 words for a 60-second script.
- 95-125 words if the visuals need more breathing room.

### Voice Tone

Use:

- Warm confidence.
- Curiosity.
- Simple emphasis on key moments.

Avoid:

- Shouting.
- Overdramatic trailer voice.
- Monotone AI delivery.
- Betting-tip certainty.

### Background Music

Style:

- Modern sports bed.
- Low percussion.
- Clean pulse.
- Subtle tension.

Rules:

- Music must sit under the voice.
- Do not use vocals.
- Do not overpower captions or data moments.
- Slight lift during dashboard and CTA.

### Sound Effects

Use:

- Intro sting.
- Soft whoosh for scene transitions.
- Tiny hit for surprising fact.
- Bar-fill sound for dashboard.
- Soft click/pulse for CTA.

Avoid:

- Constant whooshes.
- Cartoon sounds.
- Loud impacts.
- Effects that make the video feel like a betting ad.

- The first 5 seconds are boring.
- The central question is unclear.
- There is too much text on screen.
- The visuals do not change every 3-5 seconds.
- Captions are hard to read.
- The dashboard is overloaded.
- The video feels like a betting ad.
- The voiceover sounds robotic.
- The visuals are inconsistent.
- The video exceeds 60 seconds.
- The ending does not invite comments.

## 15. MVP Visual Build Roadmap

### Phase 1: Build Static Templates in Canva or Figma

Build first:

- Brand Opening.
- Surprising Fact.
- Match Card.
- Central Question.
- Evidence Card.
- Team Comparison.
- Tactical Moment.
- Insight Dashboard.
- Final CTA.

Goal:

- Prove the visual system as static frames before animating.

Do first:

- Define safe areas.
- Create typography styles.
- Create color styles.
- Build dashboard cards.
- Test readability on phone.

### Phase 2: Convert Templates Into Motion Templates

Add:

- Slow zoom.
- Punch-in.
- Slide transition.
- Bar fill.
- Number count-up.
- Dashboard reveal.

Goal:

- Make motion repeatable, not custom every day.

### Phase 3: Connect Templates to Storyboard JSON

Map fields:

- scene_id.
- template_id.
- duration.
- text.
- logos.
- chart values.
- captions.
- audio cues.

Goal:

- Make each storyboard scene renderable by a template system.

### Phase 4: Render First Test Video Manually

Use:

- One sample package.
- Liverpool vs Arsenal example.
- Manual voiceover.
- Manual asset placement.

Goal:

- Catch readability, pacing, and dashboard issues before automation.

### Phase 5: Automate Rendering

Connect to:

- Creatomate.
- Shotstack.
- Canva template workflow.
- CapCut templates.
- After Effects templates.
- Remotion.

Goal:

- Produce daily videos from structured storyboard data.

### Phase 6: Add Analytics-Based Visual Improvements

Track:

- First 3-second retention.
- Completion rate.
- Replays.
- Comments.
- Shares.
- Dashboard drop-off.
- CTA response.

Improve:

- Hook visuals.
- Caption size.
- Dashboard timing.
- Scene pacing.
- CTA wording.

### What Should Be Done First

Start with static templates.

The first build should produce:

- 10 reusable scene designs.
- One dashboard design.
- One caption style.
- One motion language.
- One sample finished video.

Do not automate rendering until the visual style works manually.
