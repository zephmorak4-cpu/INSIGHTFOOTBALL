# INSIGHT FOOTBALL Daily Input File

Version: MVP v1.0  
Template name: `daily_input.md`  
Purpose: one clean source of truth before the AI production pipeline begins.

This document is the only manual input required before the INSIGHT FOOTBALL agents start production. Fill it once per match. Keep it factual, concise, and clearly marked when data is estimated, missing, or unverified.

---

## How To Use This File

1. Copy this file for each daily production.
2. Rename the copy using this format:

```text
YYYY-MM-DD_home-team_away-team_daily_input.md
```

3. Fill all required fields.
4. Leave Story Discovery Notes blank for the Story Hunter Agent.
5. Do not force a prediction.
6. Do not make the betting market the main story.
7. Mark sample, estimated, or unverified information clearly.

Recommended save location:

```text
daily_packages/
```

# Part G — Common Mistakes

Avoid these mistakes:

- Filling the file with too much irrelevant data.
- Treating odds as the main story.
- Writing the story yourself before the Story Hunter Agent runs.
- Leaving injuries blank instead of writing "none confirmed" or "unknown".
- Mixing two matches in one input file.
- Using unverified news as fact.
- Adding complicated tactical language that will leak into the script.
- Adding stats without explaining what they mean.
- Using copyrighted images or broadcast footage without rights.
- Forgetting platform constraints.
- Forgetting that the final video must be 60 seconds or less.
- Writing like an analyst instead of a football friend.

# Part H — Best Practices

Use simple labels:

```text
verified
estimated
sample
unknown
not available
```

Write notes like this:

```text
Liverpool have started quickly at home in recent matches.
```

Not like this:

```text
Liverpool's high-intensity vertical possession sequences create superior transition pressure.
```

Keep the input balanced:

- Include enough data for the agents to find a story.
- Do not overload the file with every stat available.
- Highlight what actually matters before kickoff.

Good daily input should help the agents find:

- A compelling question.
- One surprising insight.
- A simple explanation.
- Supporting evidence.
- A reason for viewers to join the conversation.

Best workflow:

1. Fill factual match data.
2. Add simple context.
3. Mark uncertainty clearly.
4. Leave story discovery blank.
5. Run the AI pipeline.
6. Review the story before scripting.

Final rule:

```text
The daily input should inform the story. It should not write the story.
```
